## 外部调用
- 支持调用内部skill，来查找内容，返回id，组装上下文
- 支持通过id访问笔记内容
- 支持拉起笔记GUI界面
- 支持创建笔记、更新笔记（包括标签、所属文件夹等），更新需要用户确认，先预览内容（拉起编辑页面，创建新笔记或者修改旧的笔记，用户手动选择保存或者放弃），而不是直接修改，创建不需要用户确认，但是会在标题中表明'(AI新建)'字样，方便用户定位
- 一些特殊的创建和编辑：增加提醒（创建一个提醒笔记并设置好对应时间的提醒标签）、随记（在一个名为随记的笔记中结尾添加，写入日期、星期、时间，和记录的内容）、添加待办（在一个名为待办的笔记中结尾添加，一行代办的内容），这2个专项笔记通过硬编码特殊uuid锁定，用户可以编辑、删除，但是会自动创建。这些无需用户同意，可以直接做，识别工具时优先级高于一般创建和修改，根据情况对agent选择性暴露工具，例如一些环境没有GUI，但是可以连接服务器直接编辑服务器中的数据，那么只暴露这里的直接编辑（包括创建）的工具，而不暴露需要GUI的工具
- 这样实现了外部agent响应用户查询检索笔记内容（根据用户关键词，返回笔记id，根据id获取内容，组装上下文，然后回答），按用户需求创建笔记（分析用户语义，然后创建对应笔记），按用户需求编辑笔记（分析用户语义，然后编辑对应笔记，并拉起GUI，等待用户确认），用户指明要随记、待办、提醒，则直接创建对应笔记，无需确认

## migration设计方案

- aviary 项目中所有都会从 ~/orpheus-aviary-nest 中读取内容，migration运行后在指定端口上，启动时可以自定义密码，不指定时生成随机密码，本地软件如果如果链接了远程仓库，则所有操作从远程仓库中读取和写入，否则从本地仓库中读取和写入
- 支持多个从机同时连接同一个主机，注意避免冲突
- 注意，某些软件的部分文件在本地存储，例如lark的歌曲文件，这些即使连接了远程仓库，也从本地连接，即某些软件会指明哪些从本地连接，哪些从远程连接
