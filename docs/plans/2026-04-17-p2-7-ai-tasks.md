## Final Acceptance

1. `curl -N -X POST localhost:47010/ai/chat -H 'Content-Type: application/json' -d '{"message":"append my todos"}'` streams: `tool_call(add_todo)` → `note_applied` → `tool_result` → `message` → `done`.
2. `POST /ai/chat` with a create_note prompt (source=gui default) streams `draft_ready` but does NOT write DB.
3. Same prompt with `source=external` streams `preview_ready`; `POST /ai/preview/apply` writes DB.
4. `get_reminders { status: 'pending' }` returns only pending rows (joins `reminder_status`).
5. GUI config store + api.ts reflect `max_context_chars`; `/config` PATCH round-trips.
6. `editor-store` exposes `originalFolderId`; save of an AI update draft uses `patchNote` and detects folder/content/tag conflicts at save time.
7. `just check && just test` green end-to-end.
