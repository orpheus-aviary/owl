# P1-8 回收站页面设计

> 日期：2026-04-10
> 状态：已确认

## 概述

两栏布局（导航 + 内容区），内容区顶部两个 Tab 分区 + 搜索框，支持批量操作。

## 布局

```
┌──────┬───────────────────────────────────────────────┐
│      │ [回收站] [即将清除]     🔍 [搜索内容...]  [×]   │
│      │─────────────────────────────────────────────── │
│ 导航  │ [☑ 全选]  [恢复] [删除]     共 3 条             │
│ 栏    │───────────────────────────────────────────────│
│      │ ☐ 笔记标题          删除于 04-08 14:30         │
│      │     预览文字         创建 04-01 10:00           │
│      │     [#工作] [#学习]  修改 04-07 18:20           │
│      │ ☐ 笔记标题          删除于 04-09 09:15         │
│      │     ...                                       │
└──────┴───────────────────────────────────────────────┘
```

## Tab 分区

| Tab | 名称 | trash_level | 操作 |
|-----|------|-------------|------|
| Tab 1 | 回收站 | 1 | 恢复（→正常）、删除（→Tab 2） |
| Tab 2 | 即将清除 | 2 | 恢复（→Tab 1）、永久删除 |

Tab 2 显示提示文字：「超过 30 天自动清除」（P1 硬编码）

## 列表

- 复用 `NoteListItem`，额外显示删除时间（trashedAt）
- 每行左侧增加 checkbox 用于批量选择
- 搜索框：FTS 全文搜索，与编辑页搜索一致

## 批量操作

- 工具栏：全选 checkbox + 恢复按钮 + 删除/永久删除按钮 + 选中计数
- 操作后刷新列表

## 后端 API

全部复用现有端点：
- `listNotes({ trash_level: 1|2, q })` — 列表 + 搜索
- `deleteNote(id)` — level+1（Tab 1→Tab 2）
- `restoreNote(id)` — level-1（Tab 2→Tab 1，Tab 1→正常）
- `permanentDeleteNote(id)` — 永久删除
- `batchDeleteNotes(ids)` — 批量 level+1
- `batchRestoreNotes(ids)` — 批量 level-1

## 验收标准

- [ ] Tab 1 显示 trash_level=1 的笔记
- [ ] Tab 2 显示 trash_level=2 的笔记
- [ ] 搜索框过滤当前 Tab 的笔记
- [ ] Tab 1 恢复 → 笔记回到正常列表
- [ ] Tab 1 删除 → 笔记移到 Tab 2
- [ ] Tab 2 恢复 → 笔记回到 Tab 1
- [ ] Tab 2 永久删除 → 笔记彻底消失
- [ ] 批量选择 + 批量操作
- [ ] 全选/取消全选
- [ ] Tab 2 显示自动清除提示
- [ ] `just check` 零错误
